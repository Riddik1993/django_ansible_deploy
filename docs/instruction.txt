Инструкция по работе с ANSIBLE для DJANGO

1) Создать юзера на VPS, добавить в группу sudo
2) Заполнить данные по хосту в файле hosts
3) В файле deploy-demo выставить значения всех переменных
4) Работаем по роли nginx
   4.1. переименовываем шаблон под название нашего сайта (не обязательно, но желательно)
   4.2. сверяем пути в шаблоне для статики и медиа с настройками проекта
   4.3. пробуем запустить роль

5) Работаем по роли db
   5.1.проверяем нужную версию postgres в db/main.yml
   5.2. пробуем запустить роль

6) Работаем по роли app (это само django приложение)
   6.1. проверяем еще раз переменную repositary (ссылка на гит-репозитарий)
   6.2. запускаем роль app
   6.3. заходим на vps, проверяем, все ли скачалось
   6.4. заходим в venv, делаем collectstatic и миграцию через manage.py


7) Работаем по роли gunicorn
   7.1. запускаем роль
   7.2. заходим на vps, проверяем статус гуникорна командой sudo systemctl status gunicorn
   7.3. если гуникорн не запустился можно посмотреть, в чем ошибка в файле gunicorn_error.log,
        который лежит в той же папке, где проект - исправляем ошибку, пробуем повторно запустить сайт
